# Basis Pengetahuan Desa Sanreseng Ade (KB Teks - Non Dokumen)

Catatan:
- Konten ini khusus KB teks dan sengaja berbeda dari dokumen upload (Profil, Panduan Layanan, SOP Pengaduan, dan RPJM).
- Informasi bersifat dummy untuk kebutuhan testing.

## 1) Panduan Menggunakan GovConnect (WA/Webchat)
- Mulai dengan menyebutkan kebutuhan: “layanan” atau “pengaduan”.
- Sertakan informasi inti sejak awal: nama, alamat, dan ringkasan kebutuhan.
- Gunakan perintah cepat:
	- “cek status LAY-YYYYMMDD-XXX”
	- “cek status LAP-YYYYMMDD-XXX”
	- “riwayat”

## 2) Format Pesan yang Direkomendasikan
**Layanan**
- “Saya ingin mengajukan [nama layanan]. Nama: … Alamat: … Tujuan: …”

**Pengaduan**
- “Laporan [kategori]. Lokasi: … Waktu kejadian: … Dampak: …”

## 3) Kualitas Informasi Laporan
Gunakan prinsip 5W1H agar tindak lanjut lebih cepat:
- What: masalah utama
- Where: lokasi jelas + patokan
- When: waktu kejadian
- Who: pihak terdampak (jika ada)
- Why/How: kronologi singkat

## 4) Pedoman Berkas Digital
- Format disarankan: PDF/JPG/PNG.
- Nama file ringkas: “NIK_Nama_Layanan_Tgl”.
- Pastikan foto/scan jelas dan tidak terpotong.

## 5) Status & Notifikasi
- Status umum: baru → proses → selesai/ditolak.
- Perubahan status akan dikirim melalui WA/Webchat.
- Jika data kurang, admin akan meminta perbaikan.

## 6) Etika Layanan Digital
- Hindari kata kasar/fitnah.
- Sertakan data yang benar untuk menghindari penolakan.
- Satu laporan untuk satu kejadian agar penanganan terstruktur.

## 7) Template Chat Singkat
- “Halo, saya mau ajukan layanan. Nama: … Alamat: …”
- “Saya ingin melapor [kategori]. Lokasi: … Kronologi singkat: …”
- “Cek status LAY-20260126-001”
